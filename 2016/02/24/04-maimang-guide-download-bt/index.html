Windows 和 Linux 系统连上麦芒网络，不能下载内网BT资源的解决办法

  • Linux
  • Windows
  • 教程

连接上 麦芒网络 之后，我们有时候会发现网速是上去了，可是原先内网的资源却无法访问了，比如下不了 BT 资源了。

这是什么原因呢？我们经过简单分析，不难得出结论：

  • 连通 麦芒网络 之后，服务器给本机分配了一个虚拟的IP地址，这样，本机就有两个IP地址了：一个是局域网我们自己设定的静态IP，另一个就是 麦芒网络 服务器分给的虚拟IP。
  • 虽然 麦芒网络 服务器也在内网，但是它分配的IP却是虚拟的地址池里的IP，当我们连接内网时，只有用内网的IP才能连通， 麦芒网络 服务器分给的IP是不能通内网的。
  • 至于论坛为什么能上，那是因为我们做了地址伪装，这里就不谈了。

找到了原因，我们怎么解决呢？怎么能通过 麦芒网络 连接外网的同时又不能访问局域网内的资源呢？

很简单，修改系统的路由表（route 命令）就能解决这个问题。

如果你想了解更多，请阅读下面的命令解释，以及route命令的帮助信息。

以下内容是命令解释，学有余力的同学请继续阅读。

  1. route add 10.0.0.0 mask 255.0.0.0 你的电脑设置的IP -p 这条命令的作用是添加一条静态路由表。

    • -p是为了让添加的静态路由表在系统重启后依然有效 —— 在命令行窗口中键入 route 并回车可以查看帮助文档。

    • 10.0.0.0 指定网段，mask 255.0.0.0指定子网掩码，两个合在一起就表示 10.*.*.* 的所有地址；而你的电脑设置的IP就是添加的静态路由表的网关。

    • 综上所述，这条命令的目的是使得：凡是本机与10.*.*.*网段的IP收发数据都从你的电脑设置的IP这个网关走，而不走 麦芒网络 。

    • 一般情况下，10开头的IP地址都用于内网，所以也可以总结为这条命令的目的是使得：与内网IP收发的数据走内网网关（也即你的电脑设置的IP）。

  2. route delete 10.0.0.0 这条命令没什么好说的，你要是好奇，可以在运行 route delete 10.0.0.0 命令的前后，使用 route print 命令查看路由表的变化情况。

  3. Linux 用户请自行使用 man route命令查看帮助文档，不再赘述。
